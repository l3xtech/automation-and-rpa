Documentação do LogHelper

Visão geral

LogHelper é uma classe de utilitário projetada para simplificar e padronizar o registro em log em todo o aplicativo. Ele serve como um wrapper para a funcionalidade de registro fornecida pelo CoreLog, garantindo que as mensagens de registro sejam formatadas de forma consistente e incluam informações contextuais relevantes. Ao abstrair as complexidades do sistema de registro subjacente (CoreLog e OpenTelemetry), o LogHelper permite que os desenvolvedores incorporem facilmente registros ricos e estruturados em seu código.

Características principais

      Interface de registro simplificado: fornece métodos fáceis de usar para registrar mensagens em vários níveis de gravidade (depuração, informação, aviso, erro, fatal).
      Formatação de log consistente: garante que todas as mensagens de log sigam um formato padrão, incluindo categorias, tipos e níveis de gravidade predefinidos.
      Informações contextuais: oferece suporte ao registro em log com contexto adicional, como categoria de mensagem, tipo, gravidade, descrição e ação, aprimorando o conteúdo informativo dos logs.


Integração com CoreLog

O LogHelper foi projetado para funcionar perfeitamente com o CoreLog, utilizando-o para as operações reais de registro. Essa integração garante que os logs emitidos via LogHelper se beneficiem da configuração do CoreLog, incluindo qualquer roteamento, filtragem ou formatação ali definida.

Uso

Nenhuma inicialização explícita é necessária para o LogHelper em si, pois ele depende do sistema de log do CoreLog e do OpenTelemetry, que deve ser inicializado na inicialização do aplicativo, conforme descrito na documentação do CoreLog e FileLogExporter.

Emitindo registros

Para registrar uma mensagem usando LogHelper, basta chamar um de seus métodos estáticos (debug, info, warning, error, fatal) com a mensagem e qualquer informação contextual relevante:

'''
LogHelper::info("Application started successfully",
                LogHelper::CATEGORY_FUNCIONAL,
                LogHelper::TYPE_PROCESS,
                LogHelper::SEVERITY_INFORMATION,
                "The main application loop has commenced",
                "Monitoring system health");
'''

Formato de mensagem de registro

O método format no LogHelper constrói mensagens de log usando os parâmetros fornecidos, garantindo um formato consistente e informativo em todos os logs. Embora o formato específico possa ser personalizado para atender às necessidades do aplicativo, ele normalmente inclui elementos como carimbo de data/hora, nome do host, ID de log, mensagem, categoria, tipo, gravidade, descrição e ação.

Costumização

Os desenvolvedores podem estender ou modificar o LogHelper para incluir métodos de registro adicionais, incorporar novas informações contextuais ou ajustar o formato da mensagem para melhor atender aos requisitos de sua aplicação.
